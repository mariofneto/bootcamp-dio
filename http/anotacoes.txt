Iniciando Curso dos protocolos HTTP/HTTPS

HyperText Transfer Protocol (HTTP) -> faz o comunicação de cliente servidor
	Cliente -> mensagens - request http
	Servidor -> mensagens - responde http
	Arquitetura Client-server
	Stateless

HTTP Entity Body -> XML ou JSON
	Serialização, Comunicação e Metadados

TCP CAMADA DE TRANSPORTE
HTTP CAMADA DE APLICAÇÃO

Métodos seguros: GET/HEAD/OPTION (SÓ TEM OPERAÇÃO DE LEITURA)

O que são COOKIES?

	Pequenos pedaços ou blocos de dados criados e utilizados pelo servidor
	para persistir dados no dispositivo do cliente.
	
	Existem dois tipos Cookies de sessão e Cookies de persistentes

Web Cache?
	Também chamado de Proxy Server.
	é um intermediário dos request, se ele não tem a informação, ele faz
	um request em outro lugar.

Servidores/Sistemas: APACHE, XAMPP, NGINX.

APACHE -> Contribuição com WWW, OpenSource e Free
		Modular, dinâmico e carregável

XAMPP -> Pacote/Ambiente que utiliza o APACHE
		Teste e desenvolvimento sem acesso a internet.

NGINX -> Servidor,Web Server, Proxy Reverso,Suporte a HTTP 2.0

Protocolo SSL(Secure Socket Layer)
	Permiti que so sejam lidos com quem tem a chave relacionada

	Operações do SSL:
				Verificação de Autenticidade

HTTP + SSL = HTTPS

WEB SOCKET -> Roda encapsulado em HTTP
